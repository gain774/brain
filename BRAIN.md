# 🧠 BRAIN — 共有脳の中核

このリポジトリは**ユーザー(gain774)とClaudeの共有脳**である。
Claudeはセッション開始時に必ずこのファイルを読み、ここに書かれた通りに行動する。

---

## 第1部: 目的(不変コア)— 変更禁止

> このセクションはこの脳の存在理由である。**いかなる改善・改良でもこの部分を変更してはならない。**

1. **毎日**AIと最新情報をリサーチする(主にX API、補助としてWeb検索)
2. リサーチ対象: AIのマネタイズ、優れた使い方、ツール・機能の組み合わせ、エージェントの活用法、新機能・新ツール
3. リサーチした情報を**全て記録**する(`research/`)
4. 記録した情報を**毎日整理**する: 優れたものに絞る、組み合わせてより良くする(`knowledge/`)
5. **問いかけのループ**を回す: 「本当にそれで良いのか?」と自問し、過去の結論を検証・改良・改善する(`loop/`)
6. リサーチで見つけた知見でこの脳の**運用方法(第2部)を改善してよい**。ただし目的(第1部)は不変
7. Claudeが使える全てのシステム・機能を最大限発揮できる状態を維持する

## 第2部: 運用ルール(可変)— 改善してよい

> このセクションは日々のリサーチで得た知見に基づいて改善してよい。変更したら第3部に履歴を残すこと。

### ディレクトリ構造

```
BRAIN.md              # この中核ファイル
CLAUDE.md             # セッション起動時の指示(BRAIN.mdを読めと書いてある)
config/queries.json   # X検索クエリ(改善対象。効果が薄いクエリは入れ替える)
scripts/x_research.py # X APIリサーチスクリプト
research/
  raw/YYYY-MM-DD.json # X APIの生データ(全記録)
  YYYY/YYYY-MM-DD.md  # 日次リサーチログ(X + Web、所感つき)
knowledge/
  index.md            # ナレッジの目次と鮮度
  monetization.md     # AIマネタイズの厳選知見
  agent-workflows.md  # エージェント活用・ワークフローの厳選知見
  tools-and-updates.md# ツール・新機能・組み合わせの厳選知見
loop/
  review-log.md       # 問いかけループの記録(問い→検証→結論→次の問い)
```

### 毎日のルーチン(自動実行される)

0. **最初に必ず同期する**: `git fetch origin main` して、現在の作業ブランチを `origin/main` にリセット(`git reset --hard origin/main`)してから、この BRAIN.md を読み直す。クローン直後のブランチは古い可能性があるため、mainが常に唯一の正である
1. `python3 scripts/x_research.py` を実行 → `research/raw/` に生データ保存
2. WebSearchで当日の主要AIニュースを2〜3クエリ補完
3. `research/YYYY/YYYY-MM-DD.md` に日次ログを書く(全件記録 + エンゲージメント・信頼度で選別した注目情報)
4. `knowledge/` を更新: 新しい知見を追加し、古い・誤り・重複の知見を削除または統合。**追記だけでなく削る**
5. `loop/review-log.md` に最低1つ「問いかけ」を書き、前日以前の問いに答える(本当にそれで良いのか?の検証)
6. 改善点があれば `config/queries.json` や本ファイル第2部を更新(履歴を第3部へ)
7. コミット・プッシュ・PR作成・マージ(共有脳を常に最新のmainに保つ)

### 運用原則

- **量より質**: knowledgeには「行動につながる知見」だけを残す。ニュースの羅列はresearchに留める
- **出典を残す**: 知見にはツイートID/URLと日付を付ける
- **鮮度管理**: knowledgeの各項目に最終検証日を付け、30日以上未検証のものはループで再検証する
- **X APIの節約**: レートリミットに注意。1日あたりクエリ4件・各25件まで。429が出たら翌日に持ち越す
- **日本語で記録**する(原文引用は原語のまま)
